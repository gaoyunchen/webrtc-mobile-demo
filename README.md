# WebRTC 移动端视频通话 Demo

基于 WebRTC 的移动端实时视频通话演示项目，支持局域网内两台手机直接通信。

## 功能特性

- 📷 实时视频通话
- 🎤 语音通话
- 🔄 前后摄像头切换
- 📴 挂断退出
- 🎤 静音/取消静音
- ❌ 关闭/开启本地画面

## 技术原理

```
手机A ---信令---> 服务器(电脑) ---信令---> 手机B
  |                                          |
  |<=========== P2P直连（视频流）============>|
```

- **信令交换**：通过服务器交换 SDP 和 ICE 候选信息
- **P2P通信**：视频流直接在两台设备间传输，不经过服务器

## 快速开始

### 1. 安装依赖

```bash
npm install
```

### 2. 启动服务

```bash
npm start
```

服务启动后会显示访问地址，例如：`https://10.0.12.35:3000`

### 3. 手机访问

1. 两台手机连接同一 WiFi
2. 打开浏览器访问显示的地址
3. 输入相同房间号，点击「加入」
4. 两台手机都点击「开启摄像头」
5. 自动建立连接，开始通话

## 使用说明

### 按钮功能

| 按钮 | 功能 |
|------|------|
| 加入 | 加入指定房间 |
| 开启摄像头 | 打开本地摄像头和麦克风 |
| 挂断 | 结束通话并退出房间 |
| 静音 | 静音/取消静音 |
| 翻转 | 切换前后摄像头 |
| 关画面 | 关闭/开启本地视频画面 |

### HTTPS 说明

由于浏览器安全限制，摄像头访问需要 HTTPS 环境。首次访问时：

**iOS Safari：**
1. 提示"不受信任"时，点击「显示详细信息」
2. 点击「访问此网站」
3. 确认访问

**Android Chrome：**
1. 点击「高级」
2. 点击「继续前往（不安全）」

## 局域网 vs 公网

### 局域网（当前支持）

两台手机连接同一 WiFi 即可直接通信。

### 公网（需额外配置）

需要配置 STUN/TURN 服务器：

```javascript
const config = {
    iceServers: [
        { urls: 'stun:stun.l.google.com:19302' },
        // 如需 TURN 中转服务，添加：
        // { urls: 'turn:your-server.com', username: 'user', credential: 'pass' }
    ]
};
```
